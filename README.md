# Usb микрофонная решетка

# Задача

Реализовать USB микрофонную решетку с асинхронным типом синхронизации данных

# Реализация

* MCU - CH32V307
* Dev board - CH32V307V-EVT-R1
* Частота дискретизации звука - 48 кГц

    * Y1 кварц, что нужно заменить

* для записи 4 микрофонов используем два I2S, один из которых настроен как master, а другой как slave с подсоединёнными тактирующими сигналами от master'а
* на отладочной плате CH32V307VCT6-R1 high speed девайс находится справа от ETH(не там где стоит LDO)

- Счётчик от 24.576 МГц
- Измерять интервалы между USB SOF (1 мс для FS, 125 мкс для HS)
- Вычислять требуемый feedback value
- **Проверка**:
- Запись 30+ минут — без дрейфа и сбросов буфера

## Технические заметки

### PDM vs I2S (почему CIC обязателен)
PDM микрофоны выдают 1-битный поток с очень высокой частотой (3.072 МГц).  
Без CIC-фильтра с децимацией мы получим необработанный шумоподобный сигнал.  
CIC выполняет:
- Низкочастотную фильтрацию (подавление высокочастотного шума квантования)
- Децимацию в 64 раза (до 48 кГц)
- Преобразование в 16-битный PCM

### Почему 24.576 МГц?
- 24.576 / 512 = 48 кГц (LRCLK)
- 24.576 / 8 = 3.072 МГц (BCLK для 64 бит на семпл)
- Удобно для генерации точных аудиочастот
- USB High Speed работает от 24 МГц, расхождение 2.4% — допустимо

### Feedback endpoint через таймер (уточнение)
Таймер работает от того же 24.576 МГц клока. Измеряет реальный интервал между SOF и считает, сколько аудиосемплов успел обработать CIC. Это значение отправляется хосту, который подстраивает скорость отправки данных. Без этого — неизбежны пропуски/повторы семплов.

### TinyUSB на CH32V307
Уже есть успешный опыт с WinUSB, значит:
- Прерывания USB работают
- DMA для конечных точек настроен
- Проблем с портированием не ожидается

## Возможные риски и их митигация

**Риск:** Два I2S + DMA + CIC + USB — перегрузка CPU  
**Решение:** CIC оптимизировать через lookup table или использовать аппаратный фильтр (если есть)

**Риск:** Переполнение аккумуляторов CIC  
**Решение:** Использовать 40-битные аккумуляторы (через `int64_t`), нормализовать выход

**Риск:** Джиттер на BCLK при физическом соединении master→slave  
**Решение:** Короткие дорожки, согласование импедансов, использовать буферные элементы

**Риск:** Feedback endpoint в TinyUSB для UAC2  
**Решение:** Изучить пример `usb_audio` в TinyUSB, при необходимости адаптировать

**Риск:** PLL не выдаёт точную 24.576 МГц из 8 МГц  
**Решение:** Проверить коэффициенты умножения/деления (24.576 = 8 × 1536 / 500)
